# 🔐 密鑰安全管理指南 - 蒸餾與擴寫

## 1. 極精簡的一段摘要

本指南提供密鑰安全管理策略，強調絕不在代碼中硬編碼密鑰。推薦使用 Doppler 或本地 `.env` 文件通過環境變量管理密鑰，並涵蓋密鑰設置、替換、輪換及安全最佳實踐，確保敏感信息安全。

## 2. 條列式關鍵要點

*   **絕不硬編碼與提交密鑰**：永遠不要在代碼中硬編碼 API 密鑰，並確保 `.env` 等敏感文件已加入 `.gitignore`，防止意外提交。
*   **優先使用密鑰管理服務**：推薦使用 Doppler 等專業服務，或通過環境變量安全地注入和管理密鑰。
*   **定期輪換與環境隔離**：定期輪換 API 密鑰，並為開發、測試、生產等不同環境使用獨立的密鑰。
*   **熟悉密鑰替換與洩露應對**：了解如何獲取和替換各類必需/可選密鑰，並掌握密鑰洩露後的立即撤銷、生成新密鑰及更新部署流程。

## 3. 可執行建議

1.  **立即實施密鑰遷移與 Git 清理**：審查所有現有代碼庫，將硬編碼的密鑰全部移除，並遷移至 Doppler 或本地 `.env` 文件管理。同時，確認所有 `.env` 文件已正確配置在 `.gitignore` 中，並從 Git 歷史中徹底清除任何已暴露的敏感文件。
2.  **建立並執行密鑰輪換政策**：制定明確的密鑰輪換時間表（例如，每季度一次），並在任何密鑰被懷疑洩露時，立即執行撤銷
