## DEEPWEAY-SMS 開發指令模板：蒸餾與擴寫

### 1. 極精簡的一段摘要

此模板為 DEEPWEAY-SMS 項目開發提供指令，由全棧架構師主導。項目採用 Next.js 14、Go、Supabase、Gemini AI，風格極簡，強調代碼密度、錯誤處理及 Shadcn/UI。輸出要求直接提供代碼、路徑、依賴與測試。

### 2. 條列式關鍵要點

*   **項目與角色：** DEEPWEAY-SMS 項目，由全棧架構師負責開發。
*   **核心技術棧：** Next.js 14 (前端)、Go (後端)、Supabase (數據庫/認證)、Gemini AI (AI功能)。
*   **設計風格：** 極簡黑客風格，參考 GitHub Dark Mode。
*   **開發原則：** 優先考慮代碼密度、避免冗餘；前端使用 TypeScript，後端使用 Go；所有 API 調用必須包含錯誤處理；UI 組件統一使用 Shadcn/UI。
*   **輸出要求：** 直接提供實現代碼、明確的文件路徑、所需的依賴清單，以及核心功能的測試用例。

### 3. 可執行建議

1.  **建立標準化代碼生成流程：** 根據「輸出要求」，開發或採用工具鏈，確保每次代碼輸出都能嚴格遵循「直接給代碼、文件路徑、依賴清單、測試用例」的格式，提高開發效率與一致性。
2.  **實施強制性代碼審查機制：** 針對「原則」中的代碼密度、錯誤處理、TypeScript/Go 規範及 Shadcn/UI 使用，設立明確的審查標準，並將其納入 CI/CD 流程，確保所有提交都符合項目規範。
3.  **制定跨技術棧集成規範：** 由於涉及 Next.js、Go、Supabase 和 Gemini AI 多種技術，建議為各組件間的 API 接口、數據模型和錯誤碼定義清晰的集成規範，並提供示例，以減少協作摩擦和潛在的集成問題。
