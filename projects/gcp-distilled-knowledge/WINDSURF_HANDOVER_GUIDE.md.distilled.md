### 1. 極精簡的一段摘要

GCP SMS 向量噴射系統是一個基於 Node.js，利用 Gemini API 實現雙群聊定時噴射、向量切割與調試迭代的應用。本指南為接手開發者提供項目概覽、快速啟動、核心功能、開發工作流、監控與安全注意事項，以及未來規劃。

### 2. 條列式關鍵要點

*   **核心功能**: 實現雙群聊定時內容噴射，結合向量切割技術進行細緻處理與調試迭代。
*   **技術棧**: 主要採用 Node.js、Express、Gemini API，並透過 PM2 進行進程管理。
*   **快速啟動**: 需安裝 Node.js 依賴，配置 `.env` 文件中的 Gemini API Keys（區分免費與收費），並可選擇以單次、PM2 守護或 Cron 定時任務啟動服務。
*   **核心機制**: 噴射系統每小時自動執行，利用多個收費 Gemini Key 並發調用，並透過 0.1 精度向量切割處理內容，同時嚴格控制 Token 使用量。
*   **開發與部署**: 遵循 Git 工作流，日常開發後測試運行，部署時使用 PM2 停止、拉取、重啟服務。
*   **監控與日誌**: 透過 PM2 監控服務狀態與實時日誌，並有專門的日誌文件（噴射、錯誤、Cron、報告）供追蹤。
*   **安全實踐**: 強調 Key 管理（環境變量、定期輪換、不提交 Git）、日誌安全（不記錄完整 Key、定期清理）及 Token 限制監控。

### 3. 可執行建議

1.  **優先實現消息接口與平台集成**: 根據「下一步計劃」，優先開發群聊消息接口並集成 Telegram Bot，這將直接擴展系統的應用範圍和用戶互動能力，為後續功能奠定基礎。
2.  **強化數據持久化與調試能力**: 著手實現向量數據庫存儲，以提升內容管理的效率和可追溯性；同時，優化 Debug 策略，確保在複雜的向量切割和多 Key 並發場景下，能快速定位並解決問題。
3.  **提升系統可管理性與穩定性**: 規劃並開發 Web 控制面板，提供直觀的配置、監控和報告查看界面，降低運維難度；同步引入自動化測試，確保代碼質量和系統更新的穩定性。
