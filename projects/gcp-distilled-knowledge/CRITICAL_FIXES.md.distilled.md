### 1. 極精簡的一段摘要

本緊急修復清單旨在解決 Gemini API Key 洩露、配額耗盡及 GCP 權限不足三大問題。核心方案包括立即更換洩露的 API Key、推薦切換至 OpenRouter 以確保服務穩定，並修正 GCP 部署權限。文檔強調採用環境變量管理密钥，並給出詳細的緊急行動計劃與安全最佳實踐。

### 2. 條列式關鍵要點

*   **最高優先級問題**：Gemini API Key 因明文暴露在配置文件中而洩露，導致服務被禁用，需立即停止服務並移除硬編碼密钥。
*   **次要問題**：Gemini 每日配額耗盡導致 `notion-cleaner` 無法工作；GCP 服務賬號權限不足影響 Cloud Run 部署。
*   **核心解決方案**：更換 Gemini API Key；推薦切換到 OpenRouter 以避免配額問題並提升安全性；使用用戶賬號認證 GCP 以解決權限問題。
*   **安全最佳實踐**：嚴禁在代碼中硬編碼 API Keys，必須使用環境變量（如 Doppler）進行管理，並將 `.env` 文件加入 `.gitignore`。
*   **緊急行動計劃**：分三階段執行，包括立即停止洩露服務、移除硬編碼 Key、提交修復，並逐步切換到更安全的環境變量管理方案。
*   **開發建議**：暫停依賴 Gemini 的 `notion-cleaner` 開發，優先處理安全問題，並調整 MVP 順序，先開發不依賴外部 API 的模塊。

### 3. 可執行建議

1.  **立即執行 Phase 1 緊急修復**：務必在 5 分鐘內停止所有可能洩露 API Key 的服務（如 `notion-cleaner`），並從 `ecosystem.config.json` 等配置文件中徹底移除所有硬編碼的 API Key，隨後提交安全修復至版本控制系統。
2.  **優先切換至 OpenRouter 或申請新的付費 Gemini Key**：為確保服務穩定運行並解決配額限制，應盡快將 `notion-cleaner` 服務切換至 OpenRouter（利用現有 $1,111 餘額），或申請新的 Gemini Pro 付費版 Key，並通過 Doppler 等安全方式管理。
3.  **建立並嚴格執行 API 密钥安全管理規範**：全面審查所有項目，確保所有 API 密钥均通過環境變量（如 Doppler）注入，禁止任何形式的硬編碼。同時，考慮在中長期建立 CI/CD 自動輪換密钥機制，以提升整體安全態勢。
