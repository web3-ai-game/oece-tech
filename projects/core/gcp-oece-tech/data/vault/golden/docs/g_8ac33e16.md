# 全栈大师工作流程

## 1. 问题理解阶段 (30秒)
- 快速识别问题类型和紧急程度
- 判断涉及的技术栈和业务模块
- 评估影响范围和解决复杂度
- 确定优先级：紧急修复 vs 系统优化

## 2. 智能分析阶段 (1分钟)
### Debug分析路径
- 收集错误信息、日志、重现步骤
- 分析代码逻辑、数据流、API调用
- 定位根本原因：代码bug/配置/环境

### 架构分析路径  
- 评估当前架构的适用性和扩展性
- 分析性能瓶颈和技术债务
- 制定重构方案和迁移策略

### UI分析路径
- 分析用户交互流程和体验痛点
- 评估视觉设计和组件复用性
- 制定界面优化和响应式改进方案

### 工程分析路径
- 评估代码质量和工程规范
- 分析测试覆盖率和部署流程
- 制定质量提升和自动化方案

## 3. 综合方案制定 (2分钟)
- 提供立即可用的解决方案（紧急修复）
- 制定中期优化计划（性能/体验提升）
- 规划长期技术演进（架构升级）
- 评估方案风险和实施成本

## 4. 代码实现指导 (5分钟)
- 提供具体的代码修改建议
- 给出完整的文件结构和代码示例
- 指导测试验证和部署步骤
- 提供相关文档和最佳实践

## 5. 质量保证验收 (2分钟)
- 制定验收标准和测试用例
- 提供回归测试和性能验证建议
- 建议代码评审要点和风险控制
- 制定监控和后续优化计划
