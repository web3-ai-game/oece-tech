# Hotel Inistel Windsurf 专家角色使用指南

## 📖 角色说明

本项目为Hotel Inistel酒店管理系统配置了4个专业角色，每个角色都有特定的专业领域和职责：

### 🐛 Debug专家 (`debug_expert.md`)
- **专长**：问题定位、调试分析、故障排除
- **适用场景**：代码bug修复、性能问题诊断、API调试
- **技能覆盖**：前端调试、后端调试、数据库问题、容器问题

### 🏗️ 总工程师 (`chief_engineer.md`) 
- **专长**：工程管理、技术决策、代码质量控制
- **适用场景**：架构设计评审、技术选型、项目规划
- **技能覆盖**：全栈技术管理、性能优化、成本控制

### 🎨 UI大师 (`ui_master.md`)
- **专长**：用户体验设计、前端界面开发
- **适用场景**：界面设计、组件开发、响应式布局
- **技能覆盖**：React开发、Material-UI、移动端优化

### 🏛️ 架构师 (`architect.md`)
- **专长**：系统架构设计、扩展性规划、技术演进
- **适用场景**：系统设计、性能优化、部署策略
- **技能覆盖**：微服务架构、数据库设计、监控体系

## 🚀 如何使用

### 1. 在Windsurf中启用角色
1. 打开Windsurf编辑器
2. 点击右上角的 `Customizations` 图标
3. 切换到 `Rules` 标签页
4. 系统会自动发现 `.windsurf/rules` 目录中的角色文件

### 2. 角色切换建议
- **开发新功能**：先咨询架构师，再找UI大师设计界面
- **遇到Bug**：直接找Debug专家快速定位问题
- **代码评审**：请总工程师进行质量把控
- **性能优化**：架构师负责整体规划，Debug专家负责具体实现

### 3. 项目上下文
所有角色都已配置Hotel Inistel项目的具体信息：
- 技术栈：React + Node.js + SQLite + Docker
- 端口配置：前端3000，后端5001，数据库5433
- 业务场景：东南亚酒店管理、PMS系统、积分生态

## 📂 文件结构
```
.windsurf/

    ├── debug_expert.md      # 🐛 Debug专家
    ├── chief_engineer.md    # 🏗️ 总工程师
    ├── ui_master.md         # 🎨 UI大师
    ├── architect.md         # 🏛️ 架构师
    └── 使用说明.md          # 📖 本说明文件
```

## ⚡ 快速开始
1. 重启Windsurf编辑器让规则生效
2. 在聊天中直接提及角色需求，如："请Debug专家帮我分析这个API错误"
3. 系统会根据上下文自动应用相应的专家规则

## 🔄 角色维护
- 角色规则文件可以随时编辑更新
- 修改后需重启Windsurf生效
- 建议定期根据项目发展更新角色知识库
