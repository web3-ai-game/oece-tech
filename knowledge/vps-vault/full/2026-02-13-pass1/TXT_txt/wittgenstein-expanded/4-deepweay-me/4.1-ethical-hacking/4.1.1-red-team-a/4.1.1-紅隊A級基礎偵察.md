# 1. 紅隊 A級概述

紅隊 A級聚焦偵察 (Reconnaissance, MITRE TA0043)，合法 OSINT 收集目標情報，無主動掃描觸發警報。基於 NIST SP 800-115，適合初學者 Kali 環境。chunk per 工具，Firebase metadata {tool: "nmap"}，Notion code blocks 複製命令。

## 1.1 什麼是紅隊

### 1.11 紅隊（Red Team）模擬真實攻擊者進行滲透測試，驗證組織防禦（NIST SP 800-115）。

**角色對比：**
| 角色 | 重點 | 工具 |
|------|------|------|
| 紅隊 | 攻擊模擬 | Kali, Metasploit |
| 藍隊 | 防禦檢測 | ELK, Suricata |
| 紫隊 | 聯合演練 | Caldera |

### 1.12 A級範圍：純偵察階段，不觸碰漏洞利用，聚焦合法開源情報（OSINT）。

ROE (Rules of Engagement)：書面授權，scope IP/域名，NDA 保密。

### 1.13 道德框架：僅授權目標，簽 NDA/ROE，MITRE ATT&CK 框架導航。

ATT&CK Navigator：視覺化 T1592 Gather Victim Host Information。

## 1.2 Kali Linux 基礎部署

### 1.21 下載：kali.org/get-kali，選擇64-bit Installer 或 Live USB。

**硬體要求：** 4GB RAM, 50GB SSD, UEFI boot。

### 1.22 虛擬化：VMware/VirtualBox，分配4GB RAM，啟用 nested VT-x/AMD-V。

**VirtualBox 配置：**
- Network: NAT + Host-only
- Shared folder /mnt/share

### 1.23 初始化：

```bash
sudo apt update && sudo apt full-upgrade -y
sudo apt install nmap whois theharvester maltego recon-ng -y
leafpad ~/.bashrc # alias ll='ls -la'
source ~/.bashrc
```

**驗證：** `nmap --version` (7.94+)，`theHarvester -h`。

### 1.24 驗證：nmap --version 確認 7.94+。

預期輸出：Nmap version 7.94SVN ( https://nmap.org )。

## 1.3 被動偵察工具與技巧

### 1.31 域名與 IP 情報：

**whois：** `whois example.com`
輸出範例：Registrar: GoDaddy, Emails: admin@example.com, NS: ns1.bluehost。

**Shodan.io：** 免費帳號，查詢 `port:22 country:TW org:"example"` 找 SSH 伺服器。

表格：Shodan filters
| Filter | 範例 | 用途 |
|--------|------|------|
| port | port:80 | Web 服務 |
| org | org:Google | 公司資產 |
| vuln | vuln:CVE-2021-41773 | 漏洞 |

### 1.32 子域名枚舉：Sublist3r.py 或 dnsdumpster.com。

**Sublist3r：** `python3 /opt/Sublist3r/sublist3r.py -d example.com -o subs.txt`

### 1.33 人員 OSINT：theHarvester `theHarvester -d example.com -b google,linkedin,bing -l 500`。

輸出：emails.txt, hosts.txt。

## 1.4 主動偵察：Nmap 入門

### 1.41 基本語法：

```bash
nmap -sn 192.168.1.0/24 # Ping scan
nmap -sS -T4 -p- target # 全端口 SYN
nmap -sV -sC -O -p80,443 target # NSE scripts + OS
```

**輸出解析：** open ports, services version, OS guess。

### 1.42 輸出管理：-oA scan 生成 xml/gnmap/nmap。

Zenmap GUI 視覺化。

### 1.43 規避檢測：-sS --scan-delay 1s --max-rate 100。
