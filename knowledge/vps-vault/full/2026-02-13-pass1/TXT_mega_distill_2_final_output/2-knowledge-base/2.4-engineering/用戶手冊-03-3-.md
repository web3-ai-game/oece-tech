## 3. 極限燒法：3 小時 / 30 美金高效策略

此策略是針對高強度使用情境設計，背景源自敏捷開發（agile development），強調時間盒定（timeboxing）以最大化產出。原理是將任務分解為階段，控制 token 消耗（xAI API 計費基於輸入/輸出長度）。

### 3.1 階段分解與原理

1. 選材階段：精簡上下文以降低 API 成本。
2. 生成階段：使用結構化輸出確保一致性。
3. 推演階段：迭代模擬，類似蒙地卡羅方法（Monte Carlo simulation）預測故事路徑。
4. 整理階段：轉換為 JSON，便於前端渲染。

### 3.11 實例應用

在 0:30-1:30 生成「故事聖經」時，從 12 本書中提取元素，生成角色關係網。

### 3.2 成本與效率對比表

| 階段          | 時間分配     | 預估成本 (USD) | 關鍵技巧                     |
|---------------|--------------|----------------|------------------------------|
| 選材         | 30 分鐘     | 2-5           | 使用結構化命題，省 token     |
| 生成聖經     | 60 分鐘     | 10-15         | 批量提示，減少 API 呼叫      |
| 推演循環     | 60 分鐘     | 10-15         | 固定格式輸出，控制字數       |
| 整理輸出     | 30 分鐘     | 1-2           | 本地處理，無 API 依賴        |
