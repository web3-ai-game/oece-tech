## 7. 注意事項與最佳實踐

背景：版權法（copyright law）限制商用。原理：結構化輸入省 token。

### 7.1 擴展建議

先用摘要，再潤色原文。

🎯 學習路線圖

- **初級**：理解 .env 配置與基本 API 呼叫（練習 2-3 次範例代碼）。閱讀 xAI 官方文件，熟悉提示詞。
- **中級**：實作完整推演流程，生成 1-2 個世界觀。學習提示工程，優化輸出格式。
- **高級**：整合前端渲染，自定義事件模板。探索進階主題如多代理模擬（multi-agent simulation），開發自訂腳本自動化全流程。

⚡ 實戰要點
1. 始終驗證 .env key，避免 API 錯誤。
2. 控制提示詞長度，目標每回合 <300 字以省成本。
3. 使用 JSON 輸出確保結構化，便於後續處理。
4. 優先結構化素材，減少 AI hallucination。
5. 記錄每個階段輸出，方便 debug。
6. 測試多模型變異（如 grok-beta vs 其他），比較創意差異。
7. 整合版本控制（git），追蹤推演變更。
8. 定期檢查 xAI API 更新，避免端點失效。
