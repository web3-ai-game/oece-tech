# 專案目錄結構知識文檔

## 概述
本專案是一個基於 Node.js 的書籍管理或閱讀平台，包含豐富的書籍分類子目錄、資料結構定義以及伺服器端腳本。專案根目錄總計 **11 個目錄** 和 **14 個檔案**，支援書籍內容存儲、元數據管理與 Web 介面呈現。專案可能用於個人圖書館、線上閱讀系統或內容聚合平台，涵蓋多種類型中文小說（如言情、懸疑、盜墓等）。

**總計統計**：
- **目錄**：11 個
- **檔案**：14 個（包含 Markdown 文件、JSON 配置、Python/Node.js 腳本及 HTML 頁面）

## 根目錄腳本與配置文件 (5 個)
這些檔案負責專案建置、依賴管理、伺服器運行與資料處理：

- **metadata.json**：元數據配置文件，儲存專案資訊、書籍清單或 API 端點定義。

- **package.json**：Node.js 專案配置文件，定義腳本、依賴（如 Express.js）和專案資訊。
- **server.js**：主伺服器腳本，使用 Node.js 運行 Web 服務器，處理路由與靜態檔案提供。
- **structure.txt** (及 **STRUCTURE.txt**)：目錄結構文字記錄，可能為手動生成的樹狀清單，用於文檔或除錯（重複提及表示可能有大小寫變體）。

## 專案架構脈絡與使用建議
### 技術棧
- **後端**：Node.js (server.js + Express 或類似框架)。
- **前端**：靜態 HTML (public/index.html)，可擴展至 React/Vue。
- **資料處理**：Python 輔助腳本 (consolidate_books.py)。
- **內容管理**：Markdown 文檔 + JSON 元數據。

### 部署流程（推斷）
1. 安裝依賴：`npm install`。
2. 運行伺服器：`node server.js`。
3. 整理書籍：`python consolidate_books.py`。
4. 存取：瀏覽 `http://localhost/public/index.html`。

### 潛在應用
- **個人知識庫**：分類管理中文小說收藏。
- **Web 閱讀器**：透過 server.js 提供線上瀏覽。
- **內容爬取聚合**：books 子目錄暗示從網絡收集的資源。
