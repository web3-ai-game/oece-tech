## 高風險數據管理與安全重點
本層級（**敏感數據和密鑰倉庫**）專注於高危資料的生命週期管理。關鍵原則包括：

### 安全最佳實務
- **存取控制**：實施最小權限原則（PoLP），使用多因素認證（MFA）和Just-In-Time（JIT）存取。
- **加密標準**：全盤加密 + 傳輸中加密（TLS 1.3），金鑰永不暴露於記憶體。
- **監控與審計**：即時偵測異常，保留至少90天日誌。
- **風險緩解**：
  | 風險 | 緩解措施 |
  |------|----------|
  | 金鑰洩漏 | 自動輪換 + HSM儲存 |
  | 未授權存取 | 零信任架構 + 行為分析 |
  | 內部威脅 | 分離職責 + 定期審計 |
  | 供應鏈攻擊 | 軟體物料清單（SBOM）驗證 |

### 合規與法律考量
- 符合NIST SP 800-57、ISO 27001等標準。
- **警告**：高危資料洩漏可能導致嚴重後果，包括法律責任和財務損失。嚴禁在非安全環境分享。
