## 總體前提
這個LeetCode準備計劃專為軟體工程師面試設計，**每日學習時間控制在2-4小時**，確保可持續性。主要焦點是**Grokking（演算法模式理解）+ LeetCode（實戰練習）**，幫助使用者從基礎到高級逐步掌握Coding面試核心技能。

### 通用提示與資源
- **進度追蹤**：使用[Notion](https://www.notion.so/)建立個人Dashboard，記錄每日LeetCode題目、Grokking章節進度及mock面試心得。
- **mock面試**：每2周進行一次，使用Pramp、Interviewing.io或NeetCode模擬題。
- **核心資源**：
  - [Educative.io](https://www.educative.io/)：Grokking系列課程（Grokking the Coding Interview、System Design等）。
  - [NeetCode.io](https://neetcode.io/)：150題精選LeetCode清單、視頻解說及路徑圖。

**建議起點**：若無DSA基礎，從NeetCode的"New Blind 75"開始；有經驗者直接進Grokking。

  | 5-8（重疊） | Python DSA + LC | 50+ Medium LC | NeetCode Python路徑 |

  | 1-2 | Coding + Python | 2h Grokking + 2 LC | Educative + NeetCode |
  | 3-4 | System + Java | 2h設計 + 2 LC（Java） | Grokking + Java DSA |

## 執行提示與最佳實踐
- **每日結構**：50分學習（課程）+ 40分LC（盲解+優化）+ 10分Review（錯誤分析）。
- **進階技巧**：
  - LC題目：優先NeetCode 150 → Blind 75 → Company標籤（e.g., Google Top）。
  - Mock重點：時間控制（45min/題）、verbalize思路。
  - 常見陷阱：忽略邊界案例、多語言Time/Space優化。
- **追蹤模板**（Notion建議）：
  ```
  - 📅 Date | LC題號 | 時間 | Pass? | 模式 | Notes
  - 🎯 Weekly Goal | Progress | Mock Score
  ```
- **調整建議**：若卡關，延長1周；每週結束Review弱點。
