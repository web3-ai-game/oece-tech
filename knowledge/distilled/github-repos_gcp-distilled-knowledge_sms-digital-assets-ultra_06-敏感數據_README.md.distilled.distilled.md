## 核心概念與重要性
### 什麼是敏感數據和密鑰倉庫？
- **敏感數據**：任何可能導致隱私洩露或系統損害的資訊，例如：
  - 密碼哈希、私鑰、令牌（tokens）。
  - 數據庫憑證、雲服務 API 金鑰。
  - 個人識別資訊（PII）或商業機密。
- **密鑰倉庫（Key Vault）**：專用系統用於安全存儲和管理這些數據，常見工具包括 HashiCorp Vault、AWS Secrets Manager、Azure Key Vault 或 Google Secret Manager。
- **高危特性**：這些數據的洩露風險極高，可能引發：
  - 財務損失（e.g., 未授權 API 呼叫）。
  - 法律後果（e.g., GDPR、HIPAA 違規）。
  - 系統性攻擊（e.g., 供應鏈攻擊）。

### 安全管理原則
- **最小權限原則**（Least Privilege）：僅授予必要訪問。
- **輪換與過期**：定期更新密鑰（e.g., 每90天）。
- **加密傳輸與存儲**：使用 TLS 1.3+ 和 AES-256。
- **審計與監控**：記錄所有訪問，整合 SIEM 系統。
- **零信任架構**：永不假設信任，驗證每一步。
