# Gemini API 速率限制指南

本文檔詳細說明 Google Gemini API 的速率限制（Rate Limits），涵蓋**免費層 (Free Tier)** 和 **收費層 (Paid Tier 1)** 的具體配額。這些限制包括每分鐘請求數 (RPM: Requests Per Minute)、每分鐘 token 數 (TPM: Tokens Per Minute) 以及每日請求數 (RPD: Requests Per Day)。配額設計目的是平衡性能、成本與公平使用，免費層適合輕量測試，收費層則針對高負載生產環境優化。

**注意**：實際配額可能因地區、帳戶狀態或 Google 更新而略有變動。建議查閱 [官方文件](https://ai.google.dev/gemini-api/docs/rate-limits) 以獲最新資訊。RPM 限制請求頻率，TPM 限制 token 吞吐量（輸入+輸出），RPD 則為每日總上限。

## 免費層 (Free Tier)
免費層適合開發者原型測試或低頻應用，配額相對保守，重點在限制高頻濫用。以下為各模型限制：

| 模型                  | RPM (每分鐘請求) | TPM (每分鐘 Token) | RPD (每日請求) | 備註                  |
|-----------------------|------------------|--------------------|----------------|-----------------------|
| **gemini-2.5-flash-lite** | 15              | 250K              | 1K            | 輕量快速模型          |
| **gemini-2.5-flash**     | 10              | 250K              | 250           | 標準快取模型          |
| **gemini-2.5-pro**       | 2               | 125K              | 50            | **超慢**，適合精準任務 |
| **gemini-2.0-flash-lite**| 30              | 1M                | 200           | 最高速免費選項        |

**脈絡補充**：
- 免費層總體偏向低 RPM（最高 30），適合個人開發或小型腳本。
- TPM 範圍 125K–1M，足以處理中等長度對話，但不適合即時高併發。
- 超過限制會觸發 429 錯誤，建議使用指數退避（exponential backoff）重試。

## 收費層 (Paid Tier 1)
收費層需啟用計費帳戶，提供大幅提升的配額，適合企業級應用、API 服務或高吞吐量場景。重點模型如下：

| 模型                     | RPM (每分鐘請求) | TPM (每分鐘 Token) | RPD (每日請求) | 備註                  |
|--------------------------|------------------|--------------------|----------------|-----------------------|
| **gemini-3-pro-preview** | 25              | 1M                | 250           | **主力模型**，平衡性能 |
| **gemini-2.5-pro**       | 150             | 2M                | 10K           | **強力**，高精度任務   |
| **gemini-2.0-flash-lite**| 4K              | 4M                | ∞             | **超高速**，無每日上限 |

**脈絡補充**：
- 收費層 RPM 高達 4K（比免費層快數百倍），TPM 達 4M，支援大規模部署如聊天機器人或批量處理。
- **∞ RPD** 表示無硬性每日上限，受整體帳戶配額與計費影響。
- 計費依 token 使用量（輸入/輸出），速率限制獨立於費用。

## 最佳實踐
1. **選擇模型**：免費用 gemini-2.0-flash-lite 起步；生產用 gemini-3-pro-preview 或 gemini-2.5-pro。
2. **優化使用**：批次請求、快取回應、監控 token 使用以避限。
3. **錯誤處理**：429 錯誤時等待 RPM 視窗重試；檢查 [配額頁面](https://console.cloud.google.com/quotas) 申請提升。
4. **升級路徑**：免費層滿載時，啟用計費即可無縫切換更高配額。
