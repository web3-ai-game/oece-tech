# 敏感憑證與配置文檔安全管理指南

**⚠️ 目錄警告：** 此文檔目錄包含高度敏感的憑證和密鑰，包括 API 金鑰、服務器配置以及 Slack 雙因素驗證 (2FA) 備份碼。**嚴禁公開分享或上傳至任何非加密存儲系統**。洩露可能導致帳戶入侵、數據外洩或財務損失。

此目錄彙整了關鍵系統的憑證和恢復資訊，用於內部運維和故障排除。所有文件均需嚴格控制存取。

## 安全最佳實踐與建議

為防止洩露，以下為強制執行的安全措施：

### 1. **存儲與存取控制**
```
❌ 禁止：
- 推送到 GitHub / GitLab 等公開或半公開倉庫
- 儲存在未加密的雲端硬碟 (e.g., Google Drive 無密碼分享)
- 透過未加密 email 或聊天工具傳輸

✅ 推薦：
- 使用專用密碼管理器：**1Password** 或 **Bitwarden** (團隊版，支持共享金庫)
- 啟用端到端加密與多因素驗證
- 設定存取權限：僅限核心 DevOps 團隊 (e.g., 角色基於 RBAC)
```

### 2. **密鑰生命週期管理**
- **定期輪換**：每 90 天輪換所有 API 金鑰及備份碼。
  - Slack 備份碼：登入 Slack > 設定 > 安全性 > 產生新備份碼。
  - API 金鑰：透過各自控制台 (e.g., AWS IAM > 存取金鑰 > 停用舊版 > 建立新版)。
- **審核日誌**：每月檢查使用記錄，偵測未授權存取。
- **最小權限原則**：僅授予必要權限 (e.g., 讀取專用金鑰，而非全域管理員)。

### 3. **導入 Notion 前注意事項**
```
🚨 危險操作警告：
1. **先刪除或加密此目錄**：
   - 使用 `git rm --cached` 移除追蹤，再 `git commit`。
   - 或將敏感文件移至 `.gitignore` 並加密上傳。
2. 手動檢查所有 Markdown 文件，移除明文憑證。
3. 在 Notion 中使用加密區塊或外部連結至 1Password。
4. 導入後立即驗證：搜尋 "key"、"token"、"secret" 等關鍵字，確保無殘留。
```

**範例加密流程**：
```

2. 產生金鑰環：`gpg --gen-key`
```

## 緊急應變程序

| 情境 | 立即動作 |
|------|----------|
| 懷疑洩露 | 1. 停用所有相關 API 金鑰<br>2. 變更 Slack 密碼並使用備份碼重設 2FA<br>3. 通知安全團隊 |
