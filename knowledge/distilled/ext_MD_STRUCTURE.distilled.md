# 項目知識文檔：中文小說資源庫結構指南

本文件基於提供的目錄結構事實，詳細描述一個專注於中文小說資源的項目架構。該項目包含文件管理、書籍分類、數據處理腳本及 Node.js 後端服務，總計 **11 個目錄** 和 **14 個文件**。項目設計用於組織、合併及提供小說資源，可能作為本地或在線圖書館服務器運行。

## 項目概述
- **目的**：管理中文小說資源，包括分類存儲、元數據追蹤及腳本自動化處理。支援小說類型如言情、恐怖、推理等，適用於個人收藏或小型服務器部署。
- **核心功能**：
  - 目錄結構定義與文檔化。
  - 書籍合併與數據整合（透過 Python 腳本）。
  - Web 服務器提供（Node.js）。
- **總計**：**11 個目錄**（包含根目錄及子目錄），**14 個文件**（涵蓋 Markdown 文檔、腳本、JSON 配置及 HTML 頁面）。
- **技術棧**：Python（數據處理）、Node.js（後端服務）、JSON（元數據）。

## books 子目錄：小說分類系統
**books/** 目錄下有 **6 個子目錄**（事實清單列 5 個專注分類 + 1 個通用），專注中文小說分類。每個子目錄內可能存儲 TXT、EPUB 或其他格式的小說文件：

| 子目錄名稱     | 描述與脈絡 |
|----------------|------------|
| **其他**      | 通用分類，存放不屬特定類型的雜項小說（如科幻、歷史等）。 |
| **古代言情**  | 古典風格言情小說，常見穿越、重生題材，受歡迎的網絡文類型。 |
| **恐怖驚悚**  | 恐怖、懸疑故事，強調心理驚悚或靈異元素。 |
| **推理懸疑**  | 偵探、謎案類小說，類似阿加莎·克里斯蒂或東野圭吾風格的中文作品。 |
| **盜墓探險**  | 探險冒險類，典型如《盜墓筆記》系列，結合考古、神話元素。 |
| **網絡小說**  | 現代網絡連載小說，涵蓋玄幻、都市等流行類型。 |

**脈絡補充**：此分類系統反映中文網絡文學生態，方便用戶按類型瀏覽。子目錄內文件可透過腳本合併成單一檔案。

## 腳本與數據文件
這些文件位於根目錄或相關子目錄，用於數據處理與結構管理（**4 個文件**）：

| 文件/目錄名稱       | 類型     | 描述 |
|---------------------|----------|------|
| **consolidate_books.py** | Python 腳本 | 核心工具，用於合併多個小說文件（如 TXT），生成完整電子書。支援批量處理 books 子目錄。 |
| **data/structures** | 數據子目錄 | 存儲結構化數據，如目錄樹或索引文件，輔助腳本運行。 |
| **metadata.json**   | JSON 文件 | 書籍元數據庫，記錄標題、作者、類型、更新日期等，供服務器查詢。 |
| **structure.txt**   | 文本文件 | 詳細目錄結構描述（根目錄版本，與根 STRUCTURE.txt 區分）。 |

**脈絡補充**：這些工具實現自動化，例如運行 `python consolidate_books.py` 可合併 books/ 中的文件，提升資源可用性。

## 使用指南
1. **安裝**：`npm install`（Node.js 部分），確保 Python 環境。
2. **運行腳本**：`python consolidate_books.py` 合併書籍。
3. **啟動服務器**：`node server.js`，訪問 public/index.html。
4. **參考文檔**：優先閱讀 README.md 和 CATALOG.md。

此結構高效、模組化，適合擴展更多小說類型或整合數據庫。若需更新，檢查 DELETION_REPORT.md 和 metadata.json。
