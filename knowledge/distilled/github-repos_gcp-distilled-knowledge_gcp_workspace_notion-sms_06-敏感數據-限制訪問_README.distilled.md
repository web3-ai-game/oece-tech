## 🔐 安全建議與最佳實踐
為防止憑證洩露，遵循以下嚴格指南：

### 核心規則
- **絕不推送至公開倉庫**：GitHub、GitLab 等公開 repo 會導致即時洩露。使用 `.gitignore` 永久排除此目錄。
- **加密存儲優先**：
  | 工具 | 推薦原因 | 使用方式 |
  |------|----------|----------|
  | **1Password** | 端到端加密、多裝置同步、自動填入 | 建立「憑證庫」資料夾，將檔案上傳為安全附件。 |
  | **Bitwarden** | 開源、免費自託管選項 | 使用「安全檔案」功能加密上傳整個目錄。 |
- **定期輪換**：
  - API 密鑰：每 90 天輪換一次，或在疑似洩露時立即更換。
  - Slack 備份碼：使用後立即從檔案移除，並產生新組。
- **訪問控制**：
  - 僅授權必要人員（例如 DevOps 團隊）。
  - 使用 RBAC（角色基礎存取控制），如 GitHub Enterprise 或 Notion 工作區權限。

### 工具整合範例

## ⚠️ 警告與導入注意事項
### 高風險警示
- **敏感數據暴露風險**：此目錄包含**生產級憑證**，洩露可能導致：
  - 財務損失（API 濫用計費）。
  - 資料外洩（伺服器存取）。
  - 帳戶劫持（Slack 團隊控制權喪失）。
- **導入 Notion 前必做**：
  1. **刪除或加密**：將檔案移至 1Password/Bitwarden，或使用 VeraCrypt 建立加密容器。
  2. **驗證清空**：運行 `git log --all --full-history -- **/*.md` 確認無敏感 commit。
  3. **替代方案**：在 Notion 中使用「加密區塊」或外部連結至密碼管理器，而非直接貼上內容。

### 常見陷阱與應對
| 情境 | 風險 | 解決方案 |
|------|------|----------|
| 意外 git push | 公開洩露 | 立即 `git reset --hard HEAD~1` + `git push --force`，並輪換所有密鑰。通知相關服務提供者。 |
| 團隊分享 | 內部洩露 | 改用共享密碼管理器，避免檔案附件。 |
| 備份忘記加密 | 雲端洩露 | 使用工具如 `gpg` 加密：`gpg -c sensitive-folder.zip`。 |
