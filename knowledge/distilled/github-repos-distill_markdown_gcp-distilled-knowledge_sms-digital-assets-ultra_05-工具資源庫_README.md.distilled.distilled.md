# 文件庫知識文檔：全面概況與使用指南

## 1. 文件庫概況

本文件庫是一個包含 **326 個文件** 的綜合性代碼與文檔存儲庫，主要聚焦於前端開發項目（特別是 React/TSX 生態）。該庫體現了完整的開發工作流，包括源碼、配置文件、文檔與依賴管理。

### 1.1 文件分佈統計
| 文件類型 | 數量 | 佔比 | 說明 |
|---------|------|------|------|
| **`.tsx`** | 172 | **52.8%** | **核心業務組件**，佔比最高，包含 React 組件、鉤子邏輯等 |
| **`.json`** | 48 | 14.7% | 配置與數據文件，包括依賴鎖定文件 |
| **`.md`** | 46 | 14.1% | 文檔與說明文件 |
| **其他** | 60 | 18.4% | 樣式、工具腳本等 |

### 1.2 核心文件識別
- **大型 `package-lock_*.json` 文件**：多個版本的 npm 依賴鎖定文件
  - **特徵**：體積龐大（MB 級別），包含完整的依賴樹
  - **影響**：佔用大量存儲空間，是優化首要目標
  - **狀態**：存在多版本冗餘（如 `package-lock_v1.json`、`package-lock_v2.json` 等）

## 2. 元數據特徵

### 2.1 核心元數據標識
所有 **核心文件** 均包含以下標準化元數據字段：

```json
{
  "temperature": 0.7,  // 內部評估指標
  "vector_signature": "vec_abc123_xyz789"  // 向量嵌入簽名
}
```

#### 元數據解釋
| 字段 | 含義 | 典型範圍 | 用途 |
|------|------|----------|------|
| **`temperature`** | **生成溫度** | 0.0 - 1.0 | 代表 AI 處理時的創造性程度（0=確定性，1=隨機性） |
| **`vector_signature`** | **向量簽名** | 唯一字符串 | 文件內容的向量嵌入標識，用於語義檢索與相似度匹配 |

**脈絡說明**：
- 這些元數據表明文件庫經過 **AI 增強處理**（如 RAG 檢索增強生成）
- 用於內部評估、內容聚類與智能檢索
